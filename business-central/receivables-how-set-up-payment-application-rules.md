# <a name="set-up-rules-for-automatic-application-of-payments"></a><span data-ttu-id="141dd-103">Konfigurere regler for automatisk udligning af betalinger</span><span class="sxs-lookup"><span data-stu-id="141dd-103">Set Up Rules for Automatic Application of Payments</span></span>
<span data-ttu-id="141dd-104">På siden **Regler for betalingsudligning** skal du konfigurere regler, der styrer, hvordan betalingstekst (på en banktransaktion) automatisk matches med tekst på åbne poster i følgende to processer:</span><span class="sxs-lookup"><span data-stu-id="141dd-104">On the **Payment Application Rules** page, you set up rules to govern how payment text (on a bank transaction) is automatically matched with text on open entries in the following two processes:</span></span>
- <span data-ttu-id="141dd-105">Udlign automatisk betalinger til deres relaterede åbne (ubetalte) fakturaer, kreditnotaer eller andre poster, når du bruger funktionen **Udlign automatisk** på siden **Betalingsudligningskladde**.</span><span class="sxs-lookup"><span data-stu-id="141dd-105">Automatically apply payments to their related open (unpaid) invoices, credit memos, or other entries when you use the **Apply Automatically** function on the **Payment Reconciliation Journal** page.</span></span> <span data-ttu-id="141dd-106">Du kan finde flere oplysninger i [Afstemme betalinger ved hjælp af automatisk udligning](receivables-how-reconcile-payments-auto-application.md).</span><span class="sxs-lookup"><span data-stu-id="141dd-106">For more information, see [Reconcile Payments Using Automatic Application](receivables-how-reconcile-payments-auto-application.md).</span></span>

- <span data-ttu-id="141dd-107">Afstem automatisk banktransaktioner med deres relaterede interne bankkontoposter, når du vælger handlingen **Afstem automatisk** på siden **Bankkontoafstemning.**</span><span class="sxs-lookup"><span data-stu-id="141dd-107">Automatically match bank transactions with their related, internal bank account ledger entries when you choose the **Match Automatically** action on the **Bank Acc. <span data-ttu-id="141dd-109">Du kan oprette nye regler for betalingsudligning ved at vælge, hvilke typer data på en betalingsafstemningskladde, der skal matche data på en eller flere åbne poster, før den tilhørende betaling udlignes automatisk på de åbne poster.</span><span class="sxs-lookup"><span data-stu-id="141dd-109">You set up new payment application rules by choosing which types of data on a payment reconciliation journal line must match with data on one or more open entries before the related payment is automatically applied to the open entries.</span></span> <span data-ttu-id="141dd-110">Kvaliteten af hver automatisk udligning vises som en værdi af **Lav** til **Høj** i feltet **Matchtillid** på siden **Betalingsudligningskladde** ifølge den betalingsudligningsregel, der blev anvendt.</span><span class="sxs-lookup"><span data-stu-id="141dd-110">The quality of each automatic application is shown as a value of **Low** to **High** in the **Match Confidence** field on the **Payment Reconciliation Journal** page according to the payment application rule that was used.</span></span>

<span data-ttu-id="141dd-111">Hver række på siden **Regler for betalingsudligning** repræsenterer en betalingsudligningsregel.</span><span class="sxs-lookup"><span data-stu-id="141dd-111">Each row on the **Payment Application Rules** page represents a payment application rule.</span></span> <span data-ttu-id="141dd-112">Regler anvendes i den rækkefølge, der er angivet af feltet **Sorteringsrækkefølge**.</span><span class="sxs-lookup"><span data-stu-id="141dd-112">Rules are applied in the order specified by the **Sorting Order** field.</span></span> <span data-ttu-id="141dd-113">Hvis flere regler bruges samtidig, bruges den højeste sorterede regels udligningstillid.</span><span class="sxs-lookup"><span data-stu-id="141dd-113">If multiple rules are used simultaneously, then the match confidence of the highest sorted rule is used.</span></span>

<span data-ttu-id="141dd-114">Den automatiske udligningsfunktion er baseret på prioriterede afstemningskriterier.</span><span class="sxs-lookup"><span data-stu-id="141dd-114">The automatic application function is based on prioritized matching criteria.</span></span> <span data-ttu-id="141dd-115">Funktionen prøver først, i prioriteret rækkefølge, at matche teksten i de fem **Relateret part**-felter på en kladdelinje med tekst på bankkontoen, navnet eller adressen på debitorer eller kreditorer med ubetalte dokumenter, der repræsenterer åbne poster.</span><span class="sxs-lookup"><span data-stu-id="141dd-115">First the function tries, in prioritized order, to match text in the five **Related-Party** fields on a journal line with text in the bank account, name, or address of customers or vendors with unpaid documents representing open entries.</span></span> <span data-ttu-id="141dd-116">Derefter forsøger funktionen at matche tekst i felterne **Transaktionstekst** og **Flere transaktionsoplysninger** på en kladdelinje med tekst i felterne **Eksternt bilagsnr.** og **Bilagsnr.** på åbne poster.</span><span class="sxs-lookup"><span data-stu-id="141dd-116">Then, the function tries to match text in the **Transaction Text** and **Additional Transaction Info** fields on a journal line with text in the **External Document No.** and **Document No.** fields on open entries.</span></span> <span data-ttu-id="141dd-117">Endelig forsøger funktionen at finde beløbet i feltet **Kontoudtogsbeløb** på en kladdelinje med beløbet på åbne poster.</span><span class="sxs-lookup"><span data-stu-id="141dd-117">Last, the function tries to match the amount in the **Statement Amount** field on a journal line with the amount on open entries.</span></span>

> [!NOTE]
> <span data-ttu-id="141dd-118">Tekstmatchning er kun muligt for tekst, der er længere end fire tegn.</span><span class="sxs-lookup"><span data-stu-id="141dd-118">Text matching is only possible for text longer than four characters.</span></span>

<span data-ttu-id="141dd-119">Udover afstemningskriterierne gælder følgende vedrørende fortegnet foran det indbetalte beløb:</span><span class="sxs-lookup"><span data-stu-id="141dd-119">In addition to the matching criteria, the following applies concerning the sign of the payment amount:</span></span>

- <span data-ttu-id="141dd-120">I forbindelse med negative beløb foretages først en afstemning mod åbne poster, der repræsenterer debitorfakturaer, og derefter mod kreditnotaer for kreditorer.</span><span class="sxs-lookup"><span data-stu-id="141dd-120">For negative amounts, a match is made first against open entries representing customer invoices and then against vendor credit memos.</span></span>
- <span data-ttu-id="141dd-121">I forbindelse med positive beløb foretages først en afstemning mod åbne poster, der repræsenterer kreditorfakturaer, og derefter mod kreditnotaer for debitorer.</span><span class="sxs-lookup"><span data-stu-id="141dd-121">For positive amounts, a match is made first against open entries representing vendor invoices and then against customer credit memos.</span></span>
