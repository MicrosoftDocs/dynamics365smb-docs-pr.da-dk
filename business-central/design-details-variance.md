# <a name="design-details-variance"></a><span data-ttu-id="a5e18-103">Designoplysninger: Afvigelse</span><span class="sxs-lookup"><span data-stu-id="a5e18-103">Design Details: Variance</span></span>
<span data-ttu-id="a5e18-104">Afvigelsen er defineret som forskellen mellem den faktiske kostpris og standardkostprisen, som beskrevet i følgende formel.</span><span class="sxs-lookup"><span data-stu-id="a5e18-104">Variance is defined as the difference between the actual cost and the standard cost, as described in the following formula.</span></span>  

 <span data-ttu-id="a5e18-105">faktiske omkostninger – standardomkostninger = afvigelse</span><span class="sxs-lookup"><span data-stu-id="a5e18-105">actual cost – standard cost = variance</span></span>  

 <span data-ttu-id="a5e18-106">Hvis de faktiske omkostninger ændres, eksempelvis fordi du bogfører et varegebyr på en senere dato, opdateres afvigelsen tilsvarende.</span><span class="sxs-lookup"><span data-stu-id="a5e18-106">If the actual cost changes, for example, because you post an item charge on a later date, then the variance is updated accordingly.</span></span>  

> [!NOTE]  
>  <span data-ttu-id="a5e18-107">Værdiregulering påvirker ikke beregningen af afvigelse, fordi værdiregulering kun ændrer lagerværdien.</span><span class="sxs-lookup"><span data-stu-id="a5e18-107">Revaluation does not affect the variance calculation, because revaluation only changes the inventory value.</span></span>

 <span data-ttu-id="a5e18-109">Følgende eksempel illustrerer, hvordan afvigelsen beregnes for købte varer.</span><span class="sxs-lookup"><span data-stu-id="a5e18-109">The following example illustrates how variance is calculated for purchased items.</span></span> <span data-ttu-id="a5e18-110">Den bygger på følgende scenario:</span><span class="sxs-lookup"><span data-stu-id="a5e18-110">It is based on the following scenario:</span></span>
