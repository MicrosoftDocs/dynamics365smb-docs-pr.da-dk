<span data-ttu-id="dc7ce-105">Hvis f.eks. en produktionsordre om at fremstille 800 meter kræver 8 kg af en komponent, bogføres derefter, når du bogfører 200 meter som afgang, 2 kg automatisk som forbrug.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-105">For example, if a production order to produce 800 meters requires 8 kg of a component, then when you post 200 meters as output, 2 kg are automatically posted as consumption.</span></span> 

<span data-ttu-id="dc7ce-106">Denne funktion er nyttig, af følgende årsager:</span><span class="sxs-lookup"><span data-stu-id="dc7ce-106">This functionality is useful for the following reasons:</span></span>  

- <span data-ttu-id="dc7ce-107">**Lagerværdi**</span><span class="sxs-lookup"><span data-stu-id="dc7ce-107">**Inventory Valuation**</span></span>

    <span data-ttu-id="dc7ce-108">Værdiposterne for afgang og forbrug oprettes parallelt, efterhånden som produktionsordren behandles.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-108">Value entries for output and consumption are created in parallel as the production order progresses.</span></span> <span data-ttu-id="dc7ce-109">Uden rutebindingskoder vil lagerværdien forøges efterhånden som afgangen bogføres, og derefter formindsk på et senere tidspunkt, når værdien af komponentforbruget bogføres sammen med den færdige produktionsordre.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-109">Without routing link codes, the inventory value will increase as output is posted and then decrease at a later point in time when the value of component consumption is posted together with the finished production order.</span></span>  
- <span data-ttu-id="dc7ce-110">**Varedisponering**</span><span class="sxs-lookup"><span data-stu-id="dc7ce-110">**Inventory Availability**</span></span>

    <span data-ttu-id="dc7ce-111">Med gradvis bogføring af forbrug, vil tilgængeligheden af komponentvarer være mere opdateret, hvilket er vigtigt for at opretholde den interne balance mellem efterspørgsel og udbud.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-111">With gradual consumption posting, the availability of component items is more up-to-date, which is important to maintain the internal balance between demand and supply.</span></span> <span data-ttu-id="dc7ce-112">Uden rutebindingskoder vil andre efterspørgsler efter komponenten mene, at den er tilgængelig, så længe der ventes på et forsinket bogføring af forbruget.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-112">Without routing link codes, other demands for the component may believe that it is available as long as it is pending a delayed consumption posting.</span></span>  
- <span data-ttu-id="dc7ce-113">**JIT (Just-in-Time)**</span><span class="sxs-lookup"><span data-stu-id="dc7ce-113">**Just-in-Time**</span></span>

    <span data-ttu-id="dc7ce-114">Med muligheden for at tilpasse produkter til debitorernes behov, kan du minimere spild ved at sikre, at arbejds- og systemændringer kun forekommer, når de er nødvendige.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-114">With the ability to customize products to customer requests, you can minimize waste by making sure that work and system changes only occur when it is necessary.</span></span>  

<span data-ttu-id="dc7ce-115">Du kan opnå dette ved at kombinere baglæns trækmetode og rutebindingskoder, så den mængde, der udtrækkes for hver operation er proportional med den faktiske afgang af den afsluttede operation.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-115">You can achieve that by combining backward flushing method and routing link codes so that the quantity that is flushed for each operation is proportional to the actual output of the finished operation.</span></span> <span data-ttu-id="dc7ce-116">For varer, der er konfigureret med baglæns trækmetode, er standardfunktionsmåden at beregne og bogføre komponentforbrug, når du ændrer status på en frigivet produktionsordre til **Afsluttet**.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-116">For items that are set up with backward flushing method, the default behavior is to calculate and post component consumption when you change the status of a released production order to **Finished**.</span></span> <span data-ttu-id="dc7ce-117">Hvis du også definerer rutebindingskoder, forekommer beregning og bogføring, når hver operation er afsluttet, og den mængde, der rent faktisk er forbrugt i handlingen, der er bogført.</span><span class="sxs-lookup"><span data-stu-id="dc7ce-117">If you also define routing link codes, then calculation and posting occurs when each operation is finished, and the quantity that was actually consumed in the operation is posted.</span></span> <span data-ttu-id="dc7ce-118">Du kan finde flere oplysninger i [Oprette ruter](production-how-to-create-routings.md).</span><span class="sxs-lookup"><span data-stu-id="dc7ce-118">For more information, see [Create Routings](production-how-to-create-routings.md).</span></span>
